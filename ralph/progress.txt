## Codebase Patterns
- Package manager: bun. Typecheck: `bunx tsc --noEmit`.
- TanStack Query v5: `refetchInterval` callback receives `query` object (`query.state.data`), not raw data.
- All query hooks in `src/hooks/queries/` follow single-responsibility pattern (one file per query).
- `queryKeys.ts` at `src/lib/queryKeys.ts` â€” add new keys here before creating query hooks.
- `queryClient.ts` at `src/lib/queryClient.ts` â€” singleton QueryClient with global defaults.
- TQ v5 object filter syntax for invalidation: `{ queryKey: [...] }` not legacy array shorthand.
- `usePipelineStatusesQuery` key includes `projectIds[]` â€” call `removeQueries({ queryKey: ['pipelineStatuses'] })` when project list changes to remove orphaned entries.
- Reducers in `PipelinesPage/` and `PipelineDetailPage/` dropped â€” replaced with TQ queries + `useState`.
